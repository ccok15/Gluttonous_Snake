# 资料区说明

这个目录专门存放非代码类的项目资料，方便后续统一管理策划、设定、数值和表格。

建议使用方式：

- `design/`
  - 放玩法策划、系统设定、关卡思路、UI 文案草案等。
- `balance/`
  - 放数值表、成长曲线、阶段目标、掉落与概率设计等。
- `spreadsheets/`
  - 放 Excel、CSV、TSV 等表格文件。

约定：

- 跟运行逻辑直接相关的资源继续放在 `data/`。
- 跟开发流程直接相关的工程文档继续放在项目根目录，例如 `DEVLOG.md`、`ARCHITECTURE.md`、`INTERFACES.md`。
- 新增策划类文件时，优先放到这个目录下，不再散落到 `output/` 或项目根目录。
